## Known acceptable limitations

- Banner schools where SUBJECT_TO_DEPT map is empty (anywhere outside USF's subject codes) → subject disambiguation silently doesn't help, falls to numRatings + ambiguity flag. Same behavior as v1.1.1 at non-USF schools.
- PeopleSoft schools with heavily-customized themes that rename `MTG_INSTR$` cell IDs → badges silently don't render. User can report via the new bug template.
- Profs with `null` avgRating (newly listed, no ratings yet) → badge shows "—" / "—" instead of crashing. Bar is empty.
